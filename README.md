# Easy-kpop-led-animation
製作簡易應援 LED 點陣動畫(搭配開關)，可在此基礎再做更深入設計，本實作團體為ITZY

# Kpop 應援點陣燈箱
這是一個 **WeMos D1 R2 (ESP8266)** 開發的互動式應援硬體專案。結合了觸摸感應、簡易的點陣屏動畫與 OLED 顯示。

## 功能
- **互動觸摸**：使用 TTP223 電容式觸摸傳感器，一鍵切換播放/暫停。
- **雙螢幕同步**：
  - **MAX7219 8x8 點陣屏**：顯示自定義字體與動畫。
  - **OLED 螢幕**：顯示對應文字與應援語。

## 硬體清單
- **微控制器**: WeMos D1 R2 (ESP8266)
- **顯示器**: 
  - MAX7219 8x8 LED Matrix
  - 0.96" OLED Display (I2C, SSD1306)
- **傳感器**: TTP223 電容式觸摸模組
- **電子零件**: 10kΩ 電阻 (用於訊號隔離與保護)

## 接線(Wiring)
為避免 ESP8266 啟動時的腳位衝突，選用了 **D0 (GPIO 16)** 作為觸摸輸入。

| 組件 | 板端腳位 | 腳位編號 | 備註 |
| :--- | :--- | :--- | :--- |
| **TTP223 VCC** | 3.3V | - | |
| **TTP223 GND** | GND | - | |
| **TTP223 Signal**| **D0** | GPIO 16 | 避開啟動腳位，防止 Device Busy |
| **MAX7219 DIN** | D7 | GPIO 13 | MOSI |
| **MAX7219 CS** | D8 | GPIO 15 | |
| **MAX7219 CLK** | D5 | GPIO 14 | SCK |
| **OLED SCL/SDA**| D1 / D2 | GPIO 5 / 4 | I2C |

## 心得
在開發過程中，我們解決了以下關鍵硬體問題：
1. **啟動鎖死**：原本接在 D3 時，因 TTP223 電位干擾導致系統進入燒錄模式，電流掉至 0.02A。改用 **D0 (GPIO 16)** 成功解決。
2. **訊號隔離**：在 Signal 腳位串聯 **10kΩ 電阻**，能有效防止開機瞬間的電位衝擊，提升系統穩定性。(後來發現沒接電阻也可以動)
3. **SPI 衝突排除**：發現 D6 (MISO) 會被硬體 SPI 接管，故將觸摸輸入移至獨立的 D0 腳位。

## 軟體安裝
1. 將 `libs/` 資料夾下的 `ssd1306.py` 上傳至 MicroPython 開發板。
2. 將 `src/main.py` 上傳。
3. 確保 `touch_sensor = Pin(16, Pin.IN)` 設定正確。
